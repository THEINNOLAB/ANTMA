# Document Writing Protocol

Use canonical docs instead of scattering duplicate notes.

- `docs/live/CURRENT-STATE.md` holds the current project state.
- `docs/live/NEXT-STEPS.md` holds the next action list.
- `docs/live/WORKLOG.md` is append-only event history.
- Product and architecture truth belongs in top-level docs under `docs/`.

Keep examples generic and privacy-safe.
