# Product Positioning

ANTMA stands for AI-Native Team Memory Architecture.

It is a framework for teams that want AI agents to remember with boundaries,
provenance, and human review.

## Category

ANTMA is a memory architecture and governance layer, not a memory backend.

Backends can store, search, embed, or graph context. ANTMA defines what kind of
memory exists, what may become durable, what counts as current truth, and how
agents should cite evidence when recalling or completing work.

## One-Line Description

A local-first memory architecture for AI-native teams, separating agent
operating memory from reusable knowledge banks.

## What It Is

- a Markdown-first memory ledger
- a schema for memory roles and status
- a promotion workflow for durable memory
- a recall priority resolver
- evidence packet tooling
- privacy scanning for shareable workspaces
- optional adapters for external memory systems

## What It Is Not

- not a hosted memory service
- not an agent runtime
- not a vector database replacement
- not a generic note app
- not a place to store private company context

## Differentiation

Most memory tools optimize storage and retrieval. ANTMA optimizes memory
trust: which memory should be believed, which memory needs review, and which
knowledge should be reused without mixing it with live operating state.
